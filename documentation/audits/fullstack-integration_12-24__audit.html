Critical Issues

P0 Quick View Missing Variant Selection

ProductQuickView modal has NO size/color selection. Users can add products without specifying variants, leading to incorrect orders. Most customers use Quick View over detail page.

File: components/product-quick-view.tsx

Fix: Add size dropdown and color swatches from VARIANT_CONFIGS, pass selected variants to addToBag()

P0 Hardcoded API URLs

Two components use hardcoded http://localhost:8000 instead of environment variable. This will break in production.

Files: shop/page.tsx:223, checkout-button.tsx:30

Fix: Replace with process.env.NEXT_PUBLIC_API_URL

P1 Product Variants Not Persisted

Size and color selection UI is complete on product detail page, but selected variants are not saved with cart items.

File: shop/[slug]/page.tsx

Fix: Extend CartItem model and API to include selected_size and selected_color

P1 Portal Dashboard Mock Data

Dashboard shows hardcoded stats (3 events, 5 announcements) instead of real user data.

File: portal/dashboard/page.tsx

Fix: Create API endpoints for user stats, upcoming events, and announcements

Backend: /api/events/registrations/ endpoint exists
