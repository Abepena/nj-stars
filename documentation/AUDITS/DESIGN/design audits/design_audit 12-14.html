3 Critical Accessibility Issues

CRITICAL

Quantity Selector Buttons Missing ARIA Labels

The minus and plus buttons use Unicode characters only. Screen readers announce "button minus" instead of "Decrease quantity". No explanation for disabled state.

File: product-quick-view.tsx:421-437

Current (No ARIA Labels)

1

Screen reader: "button minus" / "button plus"

Recommended (Proper Labels)

1

Screen reader: "Decrease quantity, minimum reached" / "Increase quantity"

Code Fix:

// Add aria-labels
<button
  aria-label={quantity <= 1 ? "Decrease quantity (minimum reached)" : "Decrease quantity"}
  disabled={quantity <= 1}
>-</button>

<span aria-live="polite" aria-atomic="true">{quantity}</span>

<button aria-label="Increase quantity">+</button>
CRITICAL

Size Selection Buttons Below Touch Target Minimum

Size selector buttons use min-w-[2.5rem] py-1.5 (40px) which is below the WCAG 2.5 minimum of 44x44px touch targets.

File: product-quick-view.tsx:398-410

Current (40px height)

36-40px - difficult to tap on mobile

Recommended (48px minimum)

48px - meets WCAG 2.5 touch target guidelines

CRITICAL

Color Picker Swatches Lack Accessible Labels

Color swatches use basic labels like "Select Red" but don't provide hex codes or visual indication of selection state for colorblind users.

File: product-quick-view.tsx:372-386

Current (28px, minimal labels)

28px swatches, no selection indicator for screen readers

Recommended (44px hit area, better labels)

44px hit area, aria-pressed for selection, hex in label

CRITICAL

Shopping Bag Count Not Announced to Screen Readers

The bag icon shows a visual badge with item count, but this information is not available to screen reader users. They only hear "Shopping bag button".

File: site-header.tsx:112-116, mobile-nav.tsx

Current (Visual Only)

Screen reader: "Shopping bag button"

Recommended (Announces Count)

Screen reader: "Shopping bag with 3 items"

CRITICAL

Modal/Sheet Components Missing Focus Management

The product QuickView modal and mobile navigation sheet don't trap focus or set initial focus to a logical element. Keyboard users can tab outside the modal.

Files: product-quick-view.tsx:279-286, mobile-nav.tsx:76-212

Required Fixes:

  • Add aria-modal="true" to modal containers
  • Set initial focus to first interactive element (close button or primary action)
  • Implement focus trap to prevent tabbing outside modal
  • Return focus to trigger element on close
